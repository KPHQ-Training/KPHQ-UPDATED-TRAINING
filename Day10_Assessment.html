Quick Review — Days 1–9

One key takeaway per day. Read these before starting.

Day 1
Prompting & AI Fundamentals
LLMs predict the next word — they don't think or know. Every output needs your judgment. Context windows set the limit on what AI can see at once.
Day 2
Prompt Blocks & Right Tool
Six prompt blocks (Role, Task, Context, Constraints, Examples, Format) turn vague asks into reliable outputs. Know when to use chat vs automation vs agent.
Day 3
Claude Deep Dive
Artifacts produce finished work. MCP connects Claude to external tools. Projects, Skills, and context engineering separate power users from casual ones.
Day 4
Automation Foundations
Every automation: trigger + action. Map it with 4 questions before building. Always QA and keep a human in the loop for anything client-facing.
Day 5
Building Automations
Build the simplest version first, test fully, then add complexity. Your prompt is the logic inside any AI automation step.
Day 6
GHL Foundations
GHL replaces a dozen tools. Agency vs sub-account is the most important concept. Contacts, pipelines, conversations — the three things every VA must own.
Day 7
GHL Automation Workflows
Workflows run GHL on autopilot. API = waiter (you ask), webhook = doorbell (it tells you). API keys are passwords — never share them in a doc or chat.
Day 8
Meetings, PM & Messaging
AI note-takers capture decisions. Claude turns transcripts into tasks. You are the judgment layer between raw AI output and what gets sent.
Day 9
Conditional Logic
Router, path, fallback, log. A robust automation handles every case including the unexpected ones. Never skip the fallback.
