Reference · AI Concepts

AI Terms & Concepts14 terms

The vocabulary behind every tool in this course. Tap any term to expand its definition, a real-world example, and what it means for you as a VA.

Why this matters

When clients, developers, or colleagues use these words, you should understand what they mean. You do not need to be able to explain them technically — you need to know when they are relevant to your work.

LLM
Large Language Model
Concept

An AI model trained on massive amounts of text to predict the next word in a sequence. That prediction process is how it generates coherent answers. Claude, ChatGPT, and Gemini are all LLMs.

VA Note Every AI tool you use in this course runs on an LLM underneath. When someone says "the model," they mean the LLM.
Key fact LLMs predict. They don't retrieve facts from a database — they generate plausible-sounding text. That's why hallucination happens.
Token
how AI counts text
Concept

The smallest unit an LLM processes. One token ≈ ¾ of a word. "Hello world" = 2 tokens. AI pricing, context windows, and speed are all measured in tokens, not words or characters.

ExampleClaude's context window of 200,000 tokens ≈ 150,000 words ≈ roughly 500 pages of text.
VA NoteYou won't count tokens manually. But if you're on a BYOK plan (paying per token), large file uploads cost more. Trim unnecessary context before pasting.
Context Window
the AI's working memory
Concept

The total amount of text an LLM can hold in view at once — your messages, its replies, uploaded files, and system prompt. Think of it as a whiteboard that fills up. When it's full, older content drops off.

VA NoteIf output starts drifting or the AI seems to "forget" instructions in a long session — start a fresh conversation and paste only what matters now.
Key factEach new conversation = a fresh context. The AI has no memory of previous chats unless you use Projects or CLAUDE.md.
Hallucination
AI inventing facts
Concept

When an AI generates confidently-stated information that is factually wrong or completely made up — names, quotes, statistics, dates, URLs. It's not lying; it's predicting what a plausible answer looks like, even when it doesn't know.

Example"The 2022 McKinsey study found that 74% of VAs..." — sounds authoritative, may be entirely invented.
VA RuleNever send AI-generated statistics, quotes, names, or dates to a client without checking against a real source. Confident ≠ correct.
System Prompt
standing instructions before the chat
Concept

A block of instructions fed to the AI before the conversation starts. It sets the role, tone, rules, and constraints that apply to every message in that session. You never see it in the chat — it runs silently underneath.

Example"You are a concise executive assistant. Never use bullet points. Keep all replies under 80 words. Never invent figures." — set once, active in every message.
VA NoteYou set system prompts in Claude Projects (the "Instructions" field) or in CLAUDE.md for Claude Code. KeyCommand's Company Playbook is effectively a shared system prompt for all agents.
RAG
Retrieval-Augmented Generation
Concept

A technique where the AI searches an external knowledge base before answering — grounding its response in real documents rather than relying on training data alone. The result is more accurate and verifiable answers.

ExampleYou upload a client's 50-page brand guide to Claude Projects. When you ask "what's our content tone?", Claude searches that file first before responding.
VA NoteUploading files to Claude Projects or the KeyCommand Knowledge Map is a simple form of RAG. You're giving AI a reliable source to search instead of guessing.
Agent
AI that takes actions, not just replies
Concept

An AI that can plan a goal into steps and then execute those steps — opening tools, reading data, writing content, sending outputs — without waiting for a new instruction between each action. An agent acts; a chatbot replies.

ExampleYou brief an agent: "Analyze this competitor's reels and draft 3 content ideas." The agent searches, reads, analyzes, and writes — all in one run.
VA NoteKeyCommand runs on a team of agents. Your job is to brief them (give the goal), review their output, and decide what ships.
Orchestrator
the agent that manages other agents
Concept

A lead agent that receives a high-level goal, breaks it into tasks, and dispatches specialist agents to handle each one. The orchestrator coordinates the work; specialists execute it.

KeyCommandKeyPlayer is the orchestrator. When you brief it in the Boardroom, it routes work to the right specialist agents automatically.
MCP
Model Context Protocol
Concept

An open standard created by Anthropic that lets AI models connect to external tools and data sources consistently. Each app publishes an MCP "menu" of what it allows; any compatible AI can read it without custom integration work.

AnalogyAPI = the menu of what an app allows. MCP = the universal translator so AI can read any menu without learning each app separately.
VA NoteYou don't configure MCP directly. When Claude Cowork or KeyCommand connects to Slack or Gmail, MCP is the plumbing making it work.
BYOK
Bring Your Own Key
Concept

A model where you supply your own API key to a platform rather than using shared capacity. You pay the AI provider (Anthropic, OpenAI) directly for usage. The platform uses your key to run AI calls on your behalf.

KeyCommandKeyCommand requires you to connect your Anthropic API key before agents will run. All AI usage bills to your Anthropic account, not KP's.
VA RuleNever share an API key in a Slack message, doc, or email. It's as sensitive as a password.
API Key
authentication token for software
Concept

A private authentication token that grants access to an AI provider's models programmatically. Different from a login password — it's meant for software, not people. One leaked key can run up significant charges.

VA RuleNever paste an API key into a chat, doc, or email. If you're given one to configure a tool, use it only in the settings field it's meant for, then close the window.
CLAUDE.md
persistent memory file for Claude Code
Concept

A special text file that Claude Code reads every time it opens a project folder. It contains persistent instructions — client rules, file-naming conventions, what not to touch. The code-level equivalent of a Project system prompt.

Example"Always write email subjects in sentence case. Never delete files — archive them to /archive. The client's brand color is #C9A84C."
VA NoteIf a developer asks you to update the CLAUDE.md file in a project, you're setting Claude Code's standing instructions for that project. Write clearly — Claude follows this literally.
Prompt Caching
reusing repeated context for speed + cost
Concept

An API-level feature that stores repeated prompt sections (like a long system prompt or uploaded file) so the AI doesn't reprocess them on every message. Reduces cost and latency for heavy repeated context.

VA NoteYou can't toggle this from Claude.ai's chat interface — it's handled at the API level. You benefit from it indirectly when tools like KeyCommand reuse a shared system prompt across many agent runs.
Fine-tuning
retraining a model on specific data
Concept

The process of taking a base AI model and retraining it on a specific dataset so it specializes for a particular task or style. Unlike prompting, which shapes what the model does — fine-tuning changes what the model is. Requires technical expertise and API access.

VA NoteAwareness only. You will not fine-tune models as a VA. But if a client asks whether you can "train AI on our brand voice" — this is what they mean, and it's an engineering project, not a VA task.
Reference · Claude Ecosystem

Claude Ecosystem4 tools

Claude is not one product — it's a family. Knowing which version to open for which job is the skill.

Claude Chat
claude.ai
Free + Pro

The browser-based chat interface. The starting point for almost everything. You type, Claude replies. Supports file uploads, artifact creation, and voice (on mobile).

Use forDrafts, quick Q&A, summarising documents, one-off tasks, learning, experimenting. If you don't know which Claude tool to use — start here.
Free limitDaily message limits apply on the free tier. Shared training account resets daily. Use Claude Pro when available.
Claude Projects
inside Claude Chat — Pro
Pro

A saved workspace inside Claude Chat. Stores a system prompt, uploaded files, and shared chat history for a specific client or recurring job. Claude arrives pre-briefed — no re-explaining needed.

Use forOngoing client work. Each client gets their own Project: brand guide, tone examples, SOPs, and context all stored there. Open it, start tasking.
VA NoteOne Project per client or recurring responsibility. Do not mix client information across Projects — data stays clean and context stays sharp.
Claude Cowork
desktop app — Pro
Pro

Anthropic's desktop app. Connects Claude to your real tools — Slack, Gmail, Google Drive, Canva — and can act across them in sequence without you switching tabs. The difference from Claude Chat: Cowork can actually use the apps, not just write about them.

Use forMulti-step tasks that span tools. "Draft the email, save it in Docs, post the summary to Slack." One instruction, Cowork executes the full chain.
VA NoteCowork requires a connected integration for each app it touches. Set up your connectors in Settings before expecting it to read your Gmail or write to Drive.
Claude Code / CLI
terminal — technical use
Free + Pro

Claude running inside a terminal window (CLI = Command Line Interface). Can read and edit actual files on your computer, run scripts, install tools, and build automations directly — not just generate text about them.

Use forBuilding automations, batch-processing files, working alongside developers, setting up persistent memory via CLAUDE.md. Not for daily chat tasks.
VA NoteYou will encounter Claude Code when working with a developer or setting up a CLAUDE.md file. You don't need to master it on Day 3 — you need to recognize it and know when to ask your tech lead.
Reference · AI Tools

Other AI Tools4 tools

Claude is your primary tool. These are the others you will encounter — when to reach for them and what they do differently.

ChatGPT
OpenAI
Free + Paid

OpenAI's general-purpose AI assistant. Strong at conversational tasks, image generation (DALL-E), web browsing, and data analysis (Advanced Data Analysis). The most widely adopted AI tool globally as of mid-2026.

Use forImage generation (free via DALL-E), quick web research, cross-checking Claude's output with a different model, generating certificate content.
Model noteLatest model as of mid-2026: GPT-4o. Free tier limits apply and are more restrictive than Claude's free tier for long-form tasks.
Gemini
Google
Free + Paid

Google's AI assistant. Best choice when working inside Google Workspace. Native integration with Docs, Sheets, Gmail, and Drive makes it the fastest option for tasks that stay inside the Google ecosystem.

Use forGoogle Docs and Gmail drafts, summarising Google Sheets data, tasks that live entirely in Google. Use Gemini in Docs sidepanel instead of copy-pasting to Claude.
Model noteLatest model as of mid-2026: Gemini 2.5 Pro (paid). Free tier available via gemini.google.com. Gemini Advanced included in Google One AI Premium.
Perplexity
real-time research AI
Free + Paid

An AI-powered search and research tool. Every answer is grounded in live web sources with citations. Unlike Claude or ChatGPT, Perplexity is built for research with real-time data — not for writing or task execution.

Use forCompetitor research, finding current statistics, checking if a claim has a real source, quick market research when you need citations.
VA NoteUse Perplexity when you need verifiable, sourced information. Use Claude when you need to write, transform, or reason. Don't use Claude for live research — it will hallucinate.
Manus
autonomous AI agent
Paid

An autonomous AI agent that can take long-running multi-step tasks and complete them end-to-end — browsing, researching, writing, and compiling — with minimal input. More autonomous than Claude Chat or Cowork.

Use forDeep research projects that take 20–40 minutes. "Research 10 competitors and compile a report" — Manus runs it unattended while you work on something else.
VA NoteAlways verify Manus output. It is more autonomous but not more accurate — hallucination still applies. It produces a lot fast; your job is to check it before using it.
Reference · Automation & CRM

Automation & CRM3 platforms

The platforms that move data and trigger actions. These are what make AI outputs reach clients, not just sit in a chat window.

Make.com
visual automation builder
Free: 1,000 ops/mo

A visual drag-and-drop automation platform. Connects hundreds of apps in multi-step workflows. More powerful than Zapier — supports conditional logic, routers, error handling, and AI steps. Covered in Days 4–5.

Use forMulti-step automations with branching (if X → do A, if Y → do B). Connecting GHL, Google Sheets, Slack, and Claude in one flow. Building content schedulers and research pipelines.
Key conceptA "scenario" = one automation. A "module" = one step inside it. Each run = one "operation." Free tier allows 1,000 operations/month.
Zapier
simple trigger-action automation
Free: 100 tasks/mo

A simpler automation tool. Best for straight two-step automations (trigger → action) when a client already uses it or the task doesn't need branching logic. Easier to set up than Make.com but less powerful.

Use forSimple trigger-action automations: form submitted → send email, new row in Sheets → notify Slack. Quick wins when a client is already on Zapier.
VA NoteIf a client has complex branching logic (router, multiple paths), push for Make.com. Zapier is cleaner but can't match Make's conditional routing.
GoHighLevel (GHL)
CRM + marketing automation
Paid

All-in-one CRM and marketing automation platform. The core platform for most KeyPlayers clients. Replaces separate tools for email, SMS, pipelines, calendars, and follow-up workflows. Covered in Days 6–7.

Use forContact management and pipelines, automated follow-up workflows and sequences, the Conversations inbox (SMS, email, DM in one place), calendar and appointment management.
Key termsContact: a person in GHL. Pipeline: the stages a deal moves through. Workflow: an automated sequence. Trigger: what starts the workflow. Action: what the workflow does.
Reference · Meetings & Comms

Meetings & Communication4 tools

The tools that capture, organize, and route information from conversations into action.

Fathom
AI meeting note-taker
Free Tier

An AI meeting note-taker that joins Zoom, Google Meet, or Teams calls automatically. Generates a summary, action items, and transcript in real-time. Recommended free-tier pick for VAs on any video call platform.

Use forCapturing client and team meetings. The transcript and action items feed directly into your AI pipeline: paste into Claude, brief KeyCommand, or log into GHL.
VA NoteAlways notify meeting attendees that a note-taker is joining. Some clients are not comfortable with it — get verbal or written approval first.
Granola
local AI note-taker
Free + Paid

An AI note-taker that runs locally on your machine. Good for in-person meetings where a bot can't join a video call. Works by enhancing your rough notes after the fact — you jot during the meeting, Granola cleans it up.

Use forIn-person meetings, calls where adding a bot is awkward, or when the client's call platform doesn't support Fathom. Back-up option for note capture.
ClickUp
project management
Free Tier

A project management platform used by many KeyPlayers clients to track tasks, projects, and team work. VAs receive assigned tasks, update statuses, log work, and communicate within client ClickUp workspaces.

Use forReceiving and managing task assignments. Logging meeting action items. Tracking project progress for clients. Updating task statuses as work moves through stages.
VA NoteNever delete tasks in a client ClickUp without explicit permission. Archive, not delete — task history is valuable for clients reviewing past work.
Slack
team messaging
Free Tier

Team messaging platform. Standard communication tool for VA-to-client and VA-to-team communication. Channels organize conversations by topic; emoji reactions serve as lightweight workflow signals.

Emoji signals✅ = done · 👀 = reviewing · 🔁 = needs revision · 🚨 = urgent · These replace back-and-forth messages in fast-moving client channels. Learn the client's conventions early.
VA RuleNever paste sensitive client data (PII, financial records, API keys) into a Slack message — even in a private DM. Slack is not encrypted end-to-end.

KeyCommand Reference10 terms

The official terminology for KeyCommand — the AI command center you operate on behalf of clients. These terms come directly from the live platform documentation.

KeyCommand
the AI command center
KP Platform

KeyPlayers' AI marketing command center. A system of specialized AI agents that VAs direct on behalf of clients. Agents draft content, research competitors, manage approvals, and run scheduled tasks — nothing ships without VA review and client approval.

VA roleBrief agents. Review their output. Approve, reject, or redirect. You are the operator — KeyCommand is the team you manage.
KeyPlayer
the orchestrator agent
KP Platform

The lead agent in KeyCommand. Takes your request from the Boardroom, plans the work, and dispatches the right specialist agents. All agent coordination runs through KeyPlayer.

VA NoteYou talk to KeyPlayer in the Boardroom. It decides which specialist to call. You don't need to route work manually — KeyPlayer does that.
Boardroom
where you talk to KeyPlayer
KP Platform

The main chat interface inside KeyCommand. Two tabs: Orchestrator (your chat with KeyPlayer) and Agent↔Agent (a live view of agents working together in real time). Also supports iMessage via LoopMessage.

Use forBriefing KeyPlayer on a task. Checking what agents are doing mid-run. Starting a conversation from your phone via iMessage.
Operator Queue
items waiting on you — on the dashboard
KP Platform

A widget on the Overview dashboard (Workbench row) that shows items waiting for your decision. Not the same as the Approvals page — the Operator Queue is a dashboard summary; the Approvals page is where you act on items.

Common errorOperator Queue ≠ Approvals page. Operator Queue = a summary widget on the dashboard. Approvals = the actual approval queue page in the nav (URL: /drafts).
Approvals
the approval queue page
KP Platform

The page in the nav (labelled "Approvals", URL: /drafts) where every agent output lands before it can go out. Nothing agents produce — no post, email, SMS, or content — ships without passing through Approvals first.

WorkflowAgent produces output → lands in Approvals → VA reviews → Approve or Reject → if Approved: Publish / Send / Confirm to execute.
VA RuleNever rubber-stamp approvals. Every item gets a real read. You are the last human check before it reaches the client or their audience.
Autonomy Gate
how much agents can do without approval
KP Platform

A setting that controls how independently agents can act. Four levels: Observe (watch only), Propose (draft everything for approval — the default), Act + Notify (auto-run approved action types), Full Auto (agents run end-to-end without approval).

VA NoteDefault is Propose. Do not change the Autonomy Gate without client instruction. Full Auto means agents post publicly with no human review — that is a client-level decision, not a VA decision.
Strategy Genes
learned behavioral rules — Pro feature
KP Platform

Short, plain-English rules that agents have learned from your approval patterns. Example: "Lead with the number" or "Keep CTAs to one line." They are injected into agent instructions once you promote them from Proposed → Active.

LifecycleProposed → Active → Retired. A Gene starts as a pattern from the Learning scoring engine. You review and promote it. Retire any Gene instantly with one click.
VA NoteYour approval decisions are the training signal. Inconsistent approvals = inconsistent output. Approving thoughtfully = agents that improve over time. This is not the same as the Knowledge Map.
Knowledge Map
what agents have stored and learned
KP Platform

The accumulated facts and data agents have stored over time — past work, uploaded files, client decisions, previous outputs. Visible at the bottom of the Overview dashboard. Distinct from Strategy Genes.

Key distinctionKnowledge Map = stored facts and data (what agents know). Strategy Genes = learned behavioral rules from approval patterns (how agents behave). Two separate systems.
Company Playbook
the brief every agent reads first
KP Platform

A short business brief built in Settings that every agent reads before doing any work. Covers: objective, audience, voice, guardrails. The shared system prompt for the entire KeyCommand workspace.

VA NoteIf agent output consistently misses the client's tone or brand — check the Company Playbook. The instructions there are the first thing agents read. Stale or vague playbooks produce stale, vague output.
Goal → Campaign → Mission
the work hierarchy
KP Platform

The hierarchy of work in KeyCommand. Goal: a verifiable outcome agents work toward. Campaign: a multi-channel push over a date range. Mission: one big request run as coordinated waves of agent work. Task: a single individual agent run.

Full hierarchyGoal → Campaign → Mission → Wave → Task. Each level is more granular. A Goal might have two Campaigns, each with several Missions, each broken into Waves and Tasks.
