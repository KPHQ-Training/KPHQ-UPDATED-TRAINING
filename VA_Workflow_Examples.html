3 Simple Workflows Every VA Can Build

No coding, no theory, just trigger → action. Each example below is something you can build, test, and publish in a single GHL sub-account in under 15 minutes. Bookmark this page; you'll come back to it.

1. New Lead Welcome 2. Appointment Reminder 3. Missed-Call Text-Back
Example 1 of 3
New Lead Welcome

The moment a new lead comes in, they get a text and an email instead of waiting hours for a human to notice them. This is the single most common workflow in GHL, and the first one most VAs ever build.

The Flow

Trigger
Tag "new-lead" added to a contact
Action
Send a welcome SMS
Wait
Wait 5 minutes
Action
Send a welcome email

How to Build It

1
Go to Automation → Workflows → Create Workflow

Choose "Start from Scratch" so you can see every piece being added.

2
Click "Add New Trigger"

Search for and select Tag Added. In the trigger's settings, choose the tag new-lead (or whichever tag this client uses to mark a fresh lead).

3
Click the + below the trigger, add "Send SMS"

Write a short, friendly message confirming you received their info. Use a merge field like {{contact.first_name}} so it feels personal.

4
Click + again, add "Wait"

Set it to wait 5 minutes. This small gap is what keeps the sequence from feeling like a robot fired two messages at once.

5
Click + again, add "Send Email"

Write a slightly longer welcome message, what to expect next, and a way to book a call if relevant.

6
Test, then publish

Add the new-lead tag to your own test contact and confirm both messages arrive correctly before turning the workflow on for real leads.

💡
Why it matters: responding to a new lead in minutes instead of hours is one of the biggest levers in sales. This workflow does it automatically, 24/7, with zero ongoing effort from you once it's live.
Example 2 of 3
Appointment Reminder

Every booked appointment gets a confirmation right away and a reminder the day before, so the client shows up. This is the easiest way to cut no-shows for almost any service business.

The Flow

Trigger
Appointment booked
Action
Send confirmation SMS immediately
Wait
Wait until 24 hours before the appointment
Action
Send reminder SMS

How to Build It

1
Create a new workflow

Same starting point: Automation → Workflows → Create Workflow → Start from Scratch.

2
Add the trigger "Appointment Status"

Set the condition to fire when status changes to Booked or Confirmed, depending on the calendar setup.

3
Add "Send SMS" for the instant confirmation

Confirm the date, time, and location or link, using the {{appointment.time}} merge field so it's always correct.

4
Add a "Wait" step set to "Until a specific time before/after an event"

Configure it to wait until 24 hours before the appointment date and time. This is different from a simple "wait 24 hours," it always lands relative to the actual appointment.

5
Add a second "Send SMS" for the reminder

Keep it short: "Reminder: your appointment is tomorrow at {{appointment.time}}. Reply C to confirm."

6
Test on a real (or fake) booking, then publish

Book a test appointment for a few minutes out, shrink the wait step temporarily to confirm timing logic, then set it back and publish.

💡
Why it matters: automated reminders are consistently one of the biggest levers for cutting no-shows for service businesses, often by a wide margin, and they require zero manual follow-up once built.
Example 3 of 3
Missed-Call Text-Back

The simplest workflow on this page: one trigger, one action. If a call goes unanswered, the caller gets an instant text instead of just a dial tone.

The Flow

Trigger
Inbound call goes unanswered
Action
Send "sorry we missed you" SMS with a booking link

How to Build It

1
Create a new workflow

Automation → Workflows → Create Workflow → Start from Scratch.

2
Add the trigger "Customer Missed Call"

This trigger type fires specifically when an inbound call to the business number isn't answered.

3
Add "Send SMS"

Write something like: "Sorry we missed your call! How can we help? Book a time here: [link]." Keep it short and immediate. The caller is texting from the moment they hang up.

4
Test by calling the business number and not picking up

Confirm the text arrives within a few seconds, then publish.

💡
Why it matters: a missed call is one of the easiest leads to lose. Most callers won't leave a voicemail or call back; they'll just call the next business on the list. This workflow catches them before they do.

Before You Publish, Any Workflow

  • Enrolled a test contact and watched the workflow run start to finish.
  • Read every message out loud, checking tone, spelling, and merge fields (no "Hi {{contact.first_name}}" showing up literally).
  • Confirmed the trigger is scoped correctly (right tag, right calendar, right pipeline), not accidentally firing for every contact.
  • Checked the timing of any Wait steps, especially time-of-day sends, so messages don't land at 3am.
  • Turned the workflow ON (built workflows are paused by default until you toggle them live).
⚠ The #1 mistake

Publishing a workflow without testing it on a fake contact first. A broken automation doesn't fail quietly. It can text the wrong message to every real contact who matches the trigger, repeatedly. Always test on yourself before you go live.
