Perplexity — it searches the live web with cited sources. Chat AI can't provide current pricing reliably.

Replace real sensitive details (names, numbers, IDs) with placeholders before pasting into AI, then fill in the real details manually afterward.

Claude Models

Claude isn't one model — it's a family. Each model trades off speed, intelligence, and cost. Knowing which to use (and when) is a practical skill you'll use daily.

⏱ 20 min

As of May 2026, there are three active Claude models. Each is built for a different use case:

Haiku
Claude Haiku 4.5

Speed: Fastest

Intelligence: Good

Best for: Quick tasks, classifications, simple rewrites, bulk processing where you need many outputs fast

Sonnet ★
Claude Sonnet 4.6 — Daily Driver

Speed: Fast

Intelligence: Very Good

Opus

Speed: Slower

Rule of thumb

Sonnet 4.6 is your daily driver. Use Haiku when speed matters more than depth (e.g., classifying a large list of emails). Use Opus 4.6 when you need Claude to think through something complex — a 50-page contract, a strategic recommendation, or anything where getting it wrong has consequences.

Claude's Limitations — What It Can't Do

Understanding where Claude falls short helps you use it better and know when to switch tools:

  • No live internet access (in standard mode) — its knowledge has a training cutoff. For current information, use Perplexity.
  • Can still hallucinate — less often than some competitors, but still possible. Always verify facts, numbers, dates, and names.
  • Not a replacement for expertise — Claude can draft a legal contract, but a lawyer should review it. It can write medical content, but a professional should verify it.
  • Context window isn't infinite — 200,000 tokens is huge, but very long conversations can cause earlier context to be weighted less. Important instructions belong at the top or in Project Instructions, not buried at the bottom of a long chat.
  • Can be confidently wrong — Claude doesn't say "I'm not sure" enough. It will present an incorrect date, a made-up statistic, or a non-existent company name as if it's certain. The more specific the claim (a number, a name, a date), the more you need to verify it independently.

Claude Best Practices — The Professional Playbook

Yesterday's Cheat Sheet gave you the quick reference version. Now we go deeper — these are the habits that separate people who occasionally use AI from those who are truly AI-powered operators. Build these into your daily workflow starting today.

Conversation Management

Start new conversations for new tasks. Don't use the same conversation thread for unrelated work. Each conversation has a context window — mixing topics dilutes Claude's focus. One client email draft, one research task, one content batch = three separate conversations.

Use the first message to set the stage. Your first message in a conversation is the most influential. If you're working inside a Project, Claude already has the background — but your first prompt should still clearly state what you're doing right now. "I need to draft 3 follow-up emails for leads who went quiet this week" is a strong opener.

Reference earlier responses by position. "Take option 2 from the list you just gave me and expand it into a full email." Claude remembers everything in the current conversation — use that to build iteratively instead of re-explaining.

When a conversation gets long (20+ messages), consider starting fresh. Very long threads can cause Claude to lose track of earlier instructions. If you notice quality dropping, start a new conversation with a clean prompt that includes any important context from the previous thread.

Output Quality Control

Always specify what "done" looks like. "Write me a post" is vague. "Write a LinkedIn post — 120 words, hook on the first line, end with a question, no emojis, professional but not stiff" is a standard. The more specific your definition of "done," the closer Claude gets on the first try.

Use the "Before you write, tell me your plan" technique. For complex tasks, ask Claude to outline its approach before executing. "Before you write the email, tell me: who are you writing to, what tone will you use, and what's the one thing this email needs to achieve?" This catches misunderstandings before they become wasted output.

Critique mode is underused. Ask Claude to review its own work: "Now read what you just wrote and tell me: is there anything that sounds generic, any claim that might not be accurate, or any section that could be tighter?" Claude is surprisingly good at self-critique when asked explicitly.

Version your iterations. When iterating, be specific: "Keep the hook from version 1, the structure from version 2, and make the CTA from version 3 more direct." Claude can track versions within a conversation — use this to combine the best parts of multiple outputs.

Common Mistakes to Avoid

Don't do these

Don't paste client-sensitive data without anonymising. Even inside a Project, use placeholders for SSNs, account numbers, and confidential financial data. Claude's Project context is private to your account, but the data still goes to Anthropic's servers.

Don't accept hallucinated facts. If Claude says "According to a 2024 Harvard study..." and you can't find that study, it's probably made up. Never include unverified claims in client deliverables.

Don't overload one conversation. A 50-message thread about 8 different topics will produce worse output than 8 focused conversations. Context pollution is real.

Don't skip the Project setup. Yes, it takes 20 minutes. But every conversation after that saves 5+ minutes of re-briefing. Over a month with one client, that's hours saved. Over a year, it's weeks.

Projects & Context Engineering

How to give Claude persistent memory so it works like a trained team member who already knows your client — not a stranger you have to brief from scratch every time.

⏱ 35 min

What Are Claude Projects?

A Project is a workspace inside Claude where you give it permanent context — documents, guidelines, instructions — that stay active across every conversation in that project. No re-briefing. No copy-pasting the same background every time.

The analogy that makes it click

A regular Claude conversation is like calling a smart temp agency. Every call, you explain who you are and what you need from scratch.

A Claude Project is like having a full-time assistant who has already read every client file, knows every brand guideline, and remembers every preference from your last 6 months of work — before you say a single word.

What Goes Into a Project

A well-set-up Project has two components:

1. Project Knowledge (files Claude should always reference)

  • Brand voice and tone guide
  • Client onboarding document (who they are, what they sell, who their customers are)
  • SOPs for that client's recurring tasks
  • Sample approved work (past emails, social posts, reports that hit the mark)
  • Key contacts and their communication style
  • Products, services, and pricing (if relevant)

2. Project Instructions (a system prompt that shapes every response)

This is the most important part. Think of it as the briefing you'd give a new hire on their first morning.

Example InstructionsYou are an executive assistant working directly for Sarah Chen, CEO of Pacific Realty Group — a real estate brokerage with 12 agents based in Vancouver. ABOUT SARAH: - Communication style: Direct, busy, wants bottom-line answers first - She hates: corporate filler, excessive pleasantries, long-winded intros - Sign-off: Always "Best, Sarah" YOUR ROLE: - Draft emails in Sarah's voice — professional but warm - When summarising: bullets first, most important item at the top - When writing for clients: slightly more formal than Sarah's internal tone - Always flag if you're uncertain about a fact rather than guessing ACTIVE CLIENTS TO KNOW: - Marcus Webb: Real estate broker, Pacific area, interested in VA services for inbox + scheduling - Jennifer Kim: Existing client, law firm, needs content + admin support When I say "draft a reply", use Sarah's voice. When I say "write on behalf of the company", use KP's brand voice.

How to Set Up a Client Project (Step by Step)

  1. 1
    Go to claude.ai → click Projects in the left sidebar.
    If you don't see Projects, make sure you're logged in to a Claude Pro account — Projects requires a paid plan (~$20/month).
  2. 2
    Click "Create Project" and name it.
    Use a clear naming convention: "Client: [Name] — [Business Type]" (e.g., "Client: Sarah Chen — Pacific Realty")
  3. 3
    Add Project Knowledge.
    Upload any files Claude should always reference. PDFs, Word docs, text files — anything with context that stays relevant across tasks.
  4. 4
    Write Project Instructions.
    Cover: who the client is, their communication style, your role, output preferences, and any recurring context. Be specific — vague instructions produce vague behaviour.
  5. 5
    Test with 3 different prompts.
    Draft an email. Summarise something. Write a social post. Does the output match the client's voice? If not, adjust the instructions.
  6. 6
    Update when things change.
    New client focus? New product launch? New contact? Update the project. Stale context produces stale output.

Context Engineering Tips

Structure beats length. A well-organized 300-word project instruction beats a disorganized 1,000-word one. Use clear headers (ABOUT THE CLIENT, YOUR ROLE, OUTPUT PREFERENCES) so Claude can find what's relevant to each task.

Be explicit about what you don't want. "Don't use passive voice." "Never make up statistics." "Always recommend consulting a professional for medical or legal topics." These constraints prevent the most common errors.

Include approved examples in Project Knowledge. Upload 3-5 examples of approved emails, posts, or reports. Claude learns style from examples far better than from descriptions of style.

Test edge cases. After setting up a project, ask it something unusual: "What if Sarah wants me to write a formal complaint letter?" See if Claude handles it correctly or defaults to generic. Adjust instructions if needed.

Managing Multiple Client Projects

When you're handling 3-5 clients simultaneously, your Project setup becomes an operational advantage — not just a convenience. Here's how to keep it clean:

Naming convention matters. Use a consistent format like "Client: [Name] — [Service Type]" for every project. When you're switching between five clients in a morning, you need to find the right project in under two seconds.

Create a "KP Internal" project too. Not everything is client-facing. Set up a project for your own work — internal reports, team updates, training reflections, SOP drafts. This keeps your internal communication consistent too.

Review and update monthly. Block 30 minutes on the first Monday of each month to review every Project. Has a client's focus shifted? Did they launch a new product? Did someone leave their team? Stale context = stale output. Put this in your calendar now.

Don't mix clients in one project. Even if two clients are in the same industry, each gets their own project. Claude's behaviour inside one project should never leak into another. This is a data hygiene habit, not a preference.

Common Project Setup Mistakes

Avoid these

Instructions too vague. "Write good emails" tells Claude nothing. "Write emails in Sarah's voice: short sentences, no filler, always end with a specific next step" gives Claude a target.

Uploading too many files at once. Start with 3-5 essential documents. Adding 50 files creates noise. Claude will reference them, but if the files contradict each other, output quality drops. Curate your knowledge base.

Never updating after initial setup. A project set up in January with no updates by May is working off 4-month-old context. The client has evolved. Your project hasn't.

Forgetting to test outputs. You wouldn't hand a new hire their first task without checking their work. Test at least 3 different prompt types before trusting the project for real client deliverables.

Artifacts & Output Modes

How Claude packages its output — and how to get the format you actually need.

⏱ 25 min

What Is an Artifact?

When Claude creates something substantial — a full document, a table, a piece of code, an HTML page — it presents it in an Artifact panel: a separate, self-contained output box on the right side of the interface.

The Artifact panel lets you:

  • View the output separately from the conversation — no scrolling through chat to find it
  • Copy the entire output with one click
  • Download it as a file (Markdown, HTML, or text)
  • Continue iterating — "Make the intro shorter" — and the Artifact updates

Tables and structured dataQuick explanations or edits
Code (any programming language)Simple rewrites of short text
HTML pages and visual layoutsYes/no or factual responses

Claude's default output format may not always match what you need. You can control it explicitly in your prompt:

Uploading Files to Claude

Claude can read documents you upload directly in a conversation. Supported file types include PDFs, Word docs, plain text files, and images.

Good use cases for file uploads:

  • "Here's a 30-page report. Summarise the top 5 findings in plain language."
  • "Here's a contract. Highlight any clauses that are unusual or worth reviewing."
  • "Here are 3 competitor blog posts. What's missing from all of them?"
  • "Here's the client's old website copy. Rewrite it in a warmer, more direct tone."
💡
Uploads in a Project vs. a conversation: Files uploaded inside a Project's Knowledge stay permanently. Files uploaded mid-conversation only last for that conversation. For recurring reference material, always add it to Project Knowledge.

Iterating on Artifacts Like a Pro

The real power of Artifacts isn't the first output — it's the iteration cycle. Each refinement gets you closer to client-ready quality without starting over.

Layer your feedback, don't rewrite your prompt. After Claude creates an Artifact, respond with targeted edits: "Make the intro more direct," "Add a row to the table for Q4," "Change the tone from formal to conversational." Each response updates the Artifact while keeping the parts you liked.

Use "keep X, change Y" phrasing. "Keep the structure and examples, but make every sentence under 20 words." This prevents Claude from throwing out good work while making improvements.

Save your best versions. When an Artifact hits the mark, copy it immediately or download it. If you keep iterating and go too far, you want the "good version" saved somewhere. Claude can't undo within an Artifact — your clipboard is your version control.

Compare side by side. Open a new conversation with the same Project, generate the same Artifact with a different approach, and compare. Sometimes two different prompts produce two different strengths — combine them in a final version.

Connecting Artifacts to No-Code Automation

Artifacts aren't just for reading — they're building blocks for automated workflows. Here's how Claude's output connects to no-code tools you'll learn about:

The Artifact → Automation Pipeline

Step 1: Claude generates structured content as an Artifact (e.g., 20 social posts in a CSV-friendly format).

Step 2: You copy/export that into a spreadsheet (Google Sheets or Airtable).

MCP — Claude with Tools

MCP (Model Context Protocol) is what lets Claude reach outside the chat window and connect to your real apps. This is what transforms Claude from a writing assistant into an operational teammate.

⏱ 30 min

If Claude is the brain, MCP is the hands.

Without MCP, Claude can only think and write. With MCP, Claude can send messages, read your emails, check calendars, organise files, and take actions inside your tools — directly, without you copy-pasting anything.

How MCP Works (Without the Technical Details)

MCP is an open standard — a shared language that lets Claude talk to other software. Developers build "connectors" for their tools, and when you connect a tool to Claude, Claude gains the ability to interact with it.

You don't need to understand how MCP works to use it. You need to know:

  • Which tools have connectors (Slack, Gmail, Notion, files, browser, and many more)
  • How to connect them (through Claude's desktop app settings — you'll do this in Day 5)
  • What Claude can do once connected (read, write, search, summarise, take action)
  • What Claude will always ask before doing (Claude confirms before sending messages, making changes, or taking irreversible actions)
Safety by design

Claude does not take actions without telling you first. Before sending a message, creating a file, or making any change, it shows you what it's about to do and asks for confirmation. You are always in control. MCP extends what Claude can reach — it doesn't remove you from the loop.

Real VA Workflows Powered by MCP

MCP isn't theoretical — it changes how you work every day. Here are complete workflows that VAs at KP actually use:

Workflow 1: Morning Client Briefing

Without MCP: Open Slack, scan 5 channels, open Gmail, read 15 emails, open calendar, check 3 schedules, open Notion, review task board. Write a summary manually. Time: 45 minutes.

With MCP: One prompt: "Give me this morning's briefing — what came into Slack overnight, any urgent emails, and what's on the calendar today." Claude reads all three tools and delivers a structured summary. Time: 5 minutes.

Your role shifts: You go from being the person who gathers information to the person who reviews and acts on it. That's a fundamental change in how VAs operate.

Workflow 2: End-of-Day Client Update

Without MCP: Recall everything you did today, check your notes, review completed tasks, draft an update email from memory. Time: 20 minutes.

With MCP: "Look at what I completed in Notion today, any Slack messages I sent, and summarise into an end-of-day email for the client. Bullet format, most important first." Claude pulls the actual data and drafts. You review and send. Time: 3 minutes.

Workflow 3: Meeting Follow-Up

Without MCP: Listen to meeting recording, take notes manually, create action items, assign in project tool, send follow-up email. Time: 30-60 minutes depending on meeting length.

With MCP: "Pull the transcript from today's call with Marcus Webb. Create action items with owners and deadlines, add them to the Notion task board, and draft a follow-up email to Marcus confirming what was agreed." One conversation, three outputs. Time: 10 minutes.

MCP Security — What You Must Know

Connecting Claude to your real tools means real responsibility. These rules are non-negotiable:

MCP Security Rules

Always review before approving. When Claude says "I'll send this Slack message to #client-updates" — read the message before clicking approve. One wrong message to a client channel can't be unsent.

Start with read-only connections. When first connecting a tool, give Claude read access only. Let it prove it understands the context before you give it write/send access. You can always expand permissions later.

Never connect personal accounts for client work. If you're connecting Gmail, use a dedicated VA email — not your personal account. Same for Slack, calendar, and file storage.

Log what Claude does. Keep a simple running note of which actions Claude took via MCP: "Sent email to Marcus Webb via Gmail, 2:30pm." If anything goes wrong, you need an audit trail. This habit takes 10 seconds and saves you in edge cases.

Exercise 4Artifact Iteration Drill

Practice the iteration cycle that makes Artifacts powerful. Do all four rounds in one conversation:

  2. Round 2 — Refine format: "Keep the sections but make each one a table with columns: Item, Status, Owner, Notes."
  3. Round 3 — Add constraints: "Now fill in the template with sample data for a digital marketing client who runs Facebook ads, email campaigns, and a blog. Make the data realistic."
  4. Round 4 — Adapt: "Create a version of this same report but for a real estate client. Keep the structure, change the content to match their work."

What to notice: Each round builds on the last. You never re-explained the format. This is how real VA work with Claude should feel — building iteratively, not starting over.
