Workflow Fundamentals

A workflow is GoHighLevel's automation builder. Once you understand its three moving parts, you can read and build almost any automation a client needs.

The whole concept in one sentence

A workflow says: "WHEN this happens (trigger), DO these things (actions), sometimes only IF certain conditions are met."

The Three Building Blocks

1
Trigger — the "when"

The event that starts the workflow. Nothing happens until a trigger fires. Common triggers:

  • A tag is added (e.g. new-lead)
  • A form or survey is submitted
  • An appointment is booked, or status changes
  • A contact replies to a message
  • An opportunity moves to a new stage
2
Actions — the "do"

What the workflow does once triggered. Actions run in order, top to bottom. Common actions:

  • Send an SMS or email
  • Wait (a delay — e.g. "wait 1 day")
  • Add or remove a tag
  • Create or move an opportunity
  • Notify the team / assign a task
  • Send data to another tool (webhook)
3
Conditions — the "only if"

Logic that branches the path. "If/else" splits let a workflow behave differently for different contacts — e.g. if the lead booked a call, send a confirmation; else, keep following up. Conditions are what make automations feel smart instead of robotic.

A Workflow You Can Picture

Here's the most common automation in all of GHL — the instant new-lead follow-up:

Trigger
Tag "new-lead" added
→
Action
Send welcome SMS
→
Action
Wait 5 minutes
→
Action
Send booking-link email

The moment a new lead is tagged, they get a text within seconds and an email with a booking link — 24/7, with zero VA effort. Speed-to-lead like this can double a business's conversion rate.

How to Build One

1
Automation → Workflows → Create Workflow

Start from a recipe template or a blank workflow.

2
Add your trigger

Click "Add New Trigger," choose the event, and set its filters (e.g. only when source = Facebook).

3
Add actions in order

Click the + to add each step. Use Wait steps to space messages out so they feel human.

4
Test before publishing

Enroll your own test contact and watch it run. Confirm every message and tag fires correctly.

5
Publish & monitor

Toggle the workflow live. Check the enrollment history daily for the first week to catch anything odd.

The #1 automation mistake

Never publish a workflow without testing it on a fake contact first. A broken automation doesn't fail quietly — it can text hundreds of real leads the wrong message, at 3am, on repeat. Test, watch it run end-to-end, then go live. (Remember Day 5: "I automated too early" is a real and costly mistake.)

VA Workflow Recipes

You don't need to invent automations from scratch. These are the proven recipes that cover the vast majority of client needs. Learn these patterns and you can build what 90% of businesses ask for.

Recipe 1: Instant Lead Follow-Up (Speed to Lead)

TriggerNew contact created / new-lead tag added
ActionsSMS within seconds → wait 5 min → email with booking link → if no booking in 1 day, follow-up text
Why it mattersResponding in under 5 minutes vs. an hour can multiply conversions. This runs 24/7.

Recipe 2: Appointment Reminders (No-Show Killer)

TriggerAppointment booked
ActionsConfirmation SMS now → reminder 24h before → reminder 1h before
Why it mattersAutomated reminders are the single biggest lever for cutting no-shows — often by half.

Recipe 3: Long-Term Nurture

TriggerLead marked "not ready" / nurture tag added
ActionsValue email every few days for weeks; if they reply or click, notify the VA and move to active
Why it mattersMost leads aren't ready today. Nurture keeps the business top-of-mind until they are.

Recipe 4: Review Request

TriggerOpportunity marked "Won"
ActionsWait 2 days → send review request with a Google link → if no review in 5 days, gentle reminder
Why it mattersMore 5-star reviews = more trust = more leads. Fully hands-off once built.

Recipe 5: Internal Notifications

TriggerHot lead replies / high-value form submitted
ActionsInstantly notify the owner/VA by SMS, email, or Slack so a human jumps in fast
Why it mattersAutomation handles the routine; humans handle the moments that need judgment. This bridges the two.

APIs & Webhooks

These two words scare a lot of VAs. They shouldn't. You don't need to be a programmer — you need to understand what they do so you can talk about them, set them up from instructions, and know when to reach for them.

What's an API? (The Restaurant Analogy)

API = the waiter

You (one app) sit at a table. The kitchen (another app) has what you want. You don't barge into the kitchen — you tell the waiter (the API) your order, and the waiter brings back exactly what you asked for. An API (Application Programming Interface) is just the messenger that lets two apps talk to each other in a structured, agreed-upon way.

When someone says "GHL has an API," they mean: other tools can ask GHL for data (like a contact) or tell GHL to do something (like create a contact) automatically, without a human clicking buttons.

Webhook vs. API — The Key Distinction

API call — you ASK (pull)
Your tool reaches out and requests something: "Hey GHL, give me this contact's details." You initiate it. Like phoning the kitchen to ask if the food is ready.
Webhook — it TELLS you (push)
GHL automatically pushes data the moment something happens: "A new lead just came in — here are their details." You don't ask; it notifies you. Like the kitchen ringing a bell when the food's up.

Simple rule: A webhook is an action inside a GHL workflow that sends data out to another tool the instant a trigger fires. It's the most common way you'll connect GHL to the outside world — and you set it up by pasting in a URL, not by coding.

What This Looks Like in Practice

In GHL
New lead form submitted
→
Webhook action
Push lead data to a URL
→
Other tool
Make / Zapier receives it & reacts

Where You'll Actually Meet APIs & Webhooks

ScenarioWhat's Happening
Connecting GHL to Make/ZapierYou paste a webhook URL into a workflow step — data flows out automatically
Setting up an API keyIn Settings, you generate a key that lets another tool securely talk to GHL
Following a setup guideA tutorial says "add this webhook URL" — now you know exactly what that means and why
TroubleshootingIf data isn't flowing, you can check whether the webhook fired and what it sent
Security: API keys are passwords

An API key gives full programmatic access to a client's account. Treat it like a password: never paste it into a public chat, never email it in plain text, never store it where others can see it. If you ever expose one by accident, tell the client immediately so it can be regenerated. This is non-negotiable professional hygiene.

MCP & AI in GHL

This is where Week 1 and Week 2 finally connect. MCP is the bridge that lets Claude reach directly into GoHighLevel — so the AI partner you mastered in Week 1 can now work inside the CRM you mastered this week.

Quick recap from Day 4 & 5

MCP (Model Context Protocol) is the standard layer that connects Claude to your tools — Slack, Gmail, files, and yes, CRMs. With an MCP connection, you can ask Claude in plain English to read or do things inside another app, and it acts — confirming before anything irreversible.

The Three Levels of AI Inside GHL

"AI in GHL" can mean three different things. Knowing which is which keeps you clear:

Level 1
GHL's built-in AI
Native features inside GHL: Conversation AI (auto-replies to leads), Content AI (writes emails/posts), and the AI Employee add-ons.
Level 2
Claude alongside GHL
You draft in Claude, paste into GHL. No connection, just copy-paste. The workhorse approach you'll use most.
Level 3
Claude connected via MCP
Claude reaches into GHL directly — reads contacts, drafts replies, updates records — from one instruction, with your approval.

Level 1: GHL's Native AI

FeatureWhat It DoesVA Role
Conversation AIAuto-responds to lead messages, books appointments, answers FAQs 24/7Train it, set guardrails, review its conversations
Content AIGenerates email and social copy inside GHLPrompt it, then edit for voice & accuracy
Workflow AI actionsAI steps inside workflows (e.g. categorise a reply, generate a response)Configure the prompt; monitor outputs

Level 3: Connecting Claude to GHL via MCP

This is the frontier. When GHL is connected to Claude through an MCP server, you can do things like:

Imagine asking ClaudePull the 5 newest leads from [client]'s GHL account. For each, draft a personalised first-touch SMS in their voice, and tell me which look highest-intent based on the form answers. Don't send anything yet — show me first.

Claude reads the CRM, drafts the work, and queues it for your review — the agentic pattern from Day 5, now pointed at GoHighLevel.

1
A connection is set up

An MCP server links Claude to GHL (often via the GHL API key or an official connector). This is usually a one-time technical setup — you may follow a guide or have it provided.

2
You ask in plain English

No code. "Show me today's unanswered conversations." "Draft replies for the hot leads."

3
Claude acts, you approve

It reads and drafts freely, but confirms before sending, updating, or deleting anything. You remain the human in the loop.

⚠
Oversight scales with power (Day 5). An AI that can act inside a live CRM is powerful — which means your review matters more, not less. Connected AI is for drafting and surfacing; sending to real leads and changing records still passes through you. Never let Conversation AI or an MCP action run unsupervised on a new client until you've watched it behave correctly.
What you actually need to know today

You don't have to build MCP connections from scratch. You need to (1) understand that the bridge exists, (2) know what becomes possible when Claude and GHL are connected, and (3) apply the same oversight discipline you already learned. The setup is technical; the judgment is yours — and the judgment is the valuable part.

Integrations

No tool does everything. The power comes from connecting GHL to the other apps a business already uses. Here's the integration landscape and how to choose the right connector.

Three Ways to Connect GHL to Other Tools

1
Native integrations

Built right into GHL — connect with a few clicks in Settings. Examples: Google/Outlook calendar sync, Stripe payments, Facebook & Instagram, Google My Business, Mailgun. Always check for a native integration first; it's the most reliable.

2
No-code connectors (Zapier & Make)

When there's no native option, Zapier or Make bridge GHL to thousands of apps using triggers and webhooks. Example: "When a deal is Won in GHL → create a project in ClickUp → post a message in Slack." This is the VA's everyday integration toolkit (from Day 5).

3
Direct API / webhook

For custom or developer-built connections, tools talk to GHL directly through its API. You generally won't build these, but you'll recognise them and can follow a guide to plug things in.

Common Integrations You'll Set Up

Connect GHL To…So That…Method
Google CalendarBookings sync both ways; no double-bookingsNative
StripePayments & invoices process inside GHLNative
Facebook/InstagramLead ads flow straight into ContactsNative
SlackHot-lead alerts post to a channelZapier/Make
ClickUp / NotionWon deals create onboarding tasksZapier/Make
Google SheetsLeads log to a sheet for reportingZapier/Make
💡
The decision order: (1) Is there a native integration? Use it. (2) No? Can Zapier or Make connect them? Use that. (3) Still no? It's a custom API job — flag it to the client or a developer. Working top-down keeps things simple and reliable.

Hands-On Lab

Today's exercises move from reading automations to designing them. Build on test contacts only — never publish an untested workflow against real leads.

Exercise 1Read a Workflow

Open an existing workflow in a sub-account (or a recipe template). Trace it out on paper as Trigger → Action → Action. Identify the trigger, every action, and any conditions. Explain in one sentence what the whole thing accomplishes.

Exercise 2Design (and Build) the Lead Follow-Up

Build Recipe 1 from scratch in a test workflow:

  1. Trigger: a tag like test-lead is added
  2. Action: send an SMS to your test contact (your own number)
  3. Action: wait 5 minutes
  4. Action: send a follow-up email

Enroll your test contact, watch it run end-to-end, and confirm you received both messages.

Exercise 3Write the Copy with Claude

Use the prompt from the Recipes section (or write your own) to have Claude draft a full appointment-reminder sequence in a client's voice. Then paste each message into the matching workflow step. Notice how you build the logic and Claude supplies the words.

Exercise 4Explain It Back

In your own words, write 2–3 sentences each for: API, webhook, and MCP. Imagine you're explaining to a client who's nervous about "tech stuff." If you can explain it simply, you understand it.

Exercise 5Map an Integration

Pick a real scenario: "When a deal is Won in GHL, the client wants a task created in their PM tool and an alert in Slack." Map it out:

  1. What's the trigger in GHL?
  2. Native, Zapier/Make, or custom API?
  3. Write the flow: Trigger → Step 1 → Step 2 → End result.
