What You'll Build

An automated pipeline that watches a Google Drive folder for approved social media graphics, uses Claude AI to write captions and QA the content, then schedules the post in Metricool — all without manual intervention.

Why this is "agentic"
This isn't a simple A→B automation. Claude makes decisions in the middle — writing copy, checking brand alignment, and determining if the content is ready to post. That decision-making loop is what makes it an agentic workflow.

Step 1: Set Up Your Google Drive Folder Structure

1A

Create the folder hierarchy

In Google Drive, create this structure:

📁 Social Media Content 📁 Drafts ← designers drop files here 📁 For Review ← team lead reviews here 📁 Approved ← ✅ THIS triggers the automation 📁 Published ← files move here after posting
1B

File naming convention

Use this naming pattern so Claude knows what the post is about:

[Platform]-[Content Pillar]-[Short Description].png Examples: IG-AI-Stack-Tool-Comparison.png FB-Hiring-5-Signs-You-Need-VA.png LI-Framework-Delegation-Matrix.png

Claude will parse this filename to understand the platform, topic, and angle — no extra metadata needed.

Step 2: Build the Zapier Zap

Trigger: New File in Google Drive Folder

2A

Configure the trigger

  1. In Zapier, click Create Zap
  2. Choose Google Drive as the trigger app
  3. Event: New File in Folder
  4. Select your "Approved" folder
  5. Test the trigger — drop a test file in the Approved folder first

Action 1: Claude AI — Generate Caption + QA

2B

Add Claude as an action step

  1. Add a new action → search for "Claude" or "Anthropic"
  2. If not available natively, use Webhooks by Zapier → POST to https://api.anthropic.com/v1/messages
  3. Set headers:
    x-api-key: [YOUR_API_KEY] anthropic-version: 2023-06-01 content-type: application/json
  4. Set the request body (see prompt below)
2C

The Claude prompt — this is the "brain"

This prompt turns Claude into your caption writer + QA agent. Paste this into the Zapier webhook body:

{ "model": "claude-sonnet-4-20250514", "max_tokens": 1024, "messages": [ { "role": "user", "content": "You are a social media manager for KeyPlayers HQ.\n\nA new file has been approved for posting.\nFilename: {{filename_from_trigger}}\nPlatform: {{parsed_platform}}\n\nTasks:\n1. WRITE a caption (max 150 words) that matches the KeyPlayers tone: confident, direct, helpful. No fluff. Speak to business owners who are overwhelmed.\n2. Include 1 clear CTA — either join Key Leaders community OR grab a free resource. Never sell directly.\n3. Add 5-8 relevant hashtags.\n4. QA CHECK: Rate the post 1-10 on brand alignment. If below 7, flag the issue.\n\nRespond in this exact JSON format:\n{\n \"caption\": \"...\",\n \"hashtags\": \"...\",\n \"cta\": \"...\",\n \"qa_score\": 8,\n \"qa_notes\": \"...\",\n \"ready_to_post\": true\n}" } ] }
Why JSON output?
By forcing Claude to respond in JSON, Zapier can parse each field separately — caption goes to Metricool's text field, hashtags get appended, and the QA score can trigger a conditional path.

Action 2: Filter — Only Post if QA Passes

2D

Add a Zapier Filter

This is the "Decide" step in the agentic loop:

  1. Add a Filter by Zapier step
  2. Condition: ready_to_post (from Claude's response) equals true
  3. If it fails, add a Path B that sends a Slack/email notification: "Post flagged by Claude QA — manual review needed"

Action 3: Metricool — Schedule the Post

2E

Connect Metricool

  1. Add Metricool as the final action (or use their API via webhook if not natively in Zapier)
  2. Map fields:
    Metricool FieldSource
    Post TextClaude's caption + hashtags
    ImageGoogle Drive file URL from trigger
    PlatformParsed from filename (IG/FB/LI)
    Schedule TimeMetricool's "best time" or set a fixed slot
2F

Move file to Published folder

Add one last action: Google Drive → Move File — move the file from "Approved" to "Published" so it doesn't get processed again.

Step 4: Test It End to End

  1. Drop a test image into the "Approved" folder (use the naming convention)
  2. Watch Zapier — check the Zap history to see each step fire
  3. Check Claude's output — is the caption on-brand? Is the QA score accurate?
  4. Check Metricool — is the post scheduled with the right image, caption, and platform?
  5. Check Drive — did the file move to "Published"?
Common Issues
Claude returns malformed JSON: Add "You must respond ONLY with valid JSON, no extra text" to your prompt.

Image doesn't attach in Metricool: Make sure the Google Drive file is shared (at least "anyone with the link can view") so Metricool can access it.

Zap triggers on non-image files: Add a Zapier Filter after the trigger that checks file extension contains ".png" or ".jpg".

Key Concepts for VAs

Concept 1: The Agent Loop

This automation follows Plan → Act → Observe → Decide. Claude is the "brain" that acts and observes. The Zapier filter is the "decide" gate. If something fails QA, the loop routes to a human — that's the fallback.

Concept 2: Structured Output

By telling Claude to respond in JSON, we turn unstructured AI output into structured data that other tools can parse. This is how you connect AI to any automation — always demand structured output.

Concept 3: Human-in-the-Loop

The QA filter is your safety net. AI doesn't post blindly — it checks its own work, and if it's unsure, a human reviews. This is the pattern for all responsible AI automation.

Concept 4: Naming = Context

The file naming convention isn't just organization — it's how you give Claude context without a database. The filename IS the prompt input. Smart naming = smart AI output.
