Day 5 · Free Tool Build

Pick Your Path

Two free options. Same outcome: a real, working automation you built yourself. Pick the one that fits you, then follow that track for the next 3 slides.

Track A · Google Apps Script
Form → Sheet → Gmail alert
  • Runs entirely inside Google
  • No third-party account needed
  • 100% free, no task limits
  • You paste ~10 lines of code
Best for: VAs already in Google Workspace
Track B · Zapier Free Tier
Form → Gmail alert (visual, no code)
  • Point-and-click, zero code
  • Free: 100 tasks/month, unlimited Zaps
  • Works with 6,000+ apps
  • Free zapier.com account required
Best for: VAs who want a no-code visual builder
Both tracks build the same thing

When a Google Form is submitted, you get an email alert with the response data. The difference is only in how you build it. Either track counts as completing today's activity.

Track A · Google Apps Script

Step 1: Set Up the Sheet and Script

Three steps before you write any code.

Setup guide
Step 1 Create a Google Form
Add 3 fields: Name (short answer), Email (short answer), Message (paragraph)

Step 2 Link to a Sheet
In the form: Responses tab > green Sheets icon > Create new spreadsheet

Step 3 Open Apps Script
In the Sheet: Extensions > Apps Script > opens editor in new tab
Note on authorization

When you run the script for the first time, Google will ask you to authorize it. Click "Advanced" then "Go to [project name] (unsafe)" and allow. This is normal for Apps Script.

Track A · Google Apps Script

Step 2: Write the Trigger Function

Paste this into the Apps Script editor, replacing any existing code. Save with Ctrl+S.

Apps Script — Code.gs
function notifyOnSubmit(e) {
// e.values = array of all form answers in order
var name = e.values[1];
var email = e.values[2];
var message = e.values[3];
MailApp.sendEmail({
to: Session.getActiveUser().getEmail(),
subject: 'New form response from ' + name,
body: 'Name: ' + name + '\n' +
'Email: ' + email + '\n' +
'Message: ' + message
});
}
Connect the trigger

In Apps Script: click the alarm-clock icon (Triggers) on the left sidebar > Add Trigger > Choose function: notifyOnSubmit > Event source: From spreadsheet > Event type: On form submit > Save.

Track A · Google Apps Script

Step 3: Test and Debug

Your turn (5 min)
Submit a test response
  1. Open your form and submit a test response with dummy data
  2. Check your Gmail inbox for the alert email
  3. In Apps Script, click Executions (left sidebar) to see the run log
Error messageCauseFix
Authorization requiredNever ran manuallyRun the function once manually from the editor to trigger the authorization flow
No email receivedSpam filter or wrong addressCheck spam folder; verify the to: address resolves to your inbox

Track B · Zapier Free Tier

Step 1: Create Your Zap

A Zap is a single automation in Zapier. Free tier gives you 100 tasks/month and unlimited single-step Zaps.

Zapier setup
Step 1 Create a free account
Go to zapier.com > Sign up with Google > free plan

Step 2 Create a Google Form
Same as Track A: 3 fields — Name, Email, Message. Link to a Sheet.

Step 3 Start a new Zap
In Zapier: click "Create" > "Zap" > opens the visual builder

Step 4 Set your trigger app
Search "Google Forms" > Event: "New Form Response" > connect account
Zapier needs the linked Sheet, not the Form

When connecting Google Forms in Zapier, you select the spreadsheet that your form responses log to, not the form itself. Make sure you created the linked Sheet first (Responses tab > green Sheets icon).

Track B · Zapier Free Tier

Step 2: Set the Action and Turn It On

Once the trigger is connected, add the email action, test it, and activate.

Zapier build — continued
Step 5 Add an action
Click the "+" below the trigger > search "Gmail" > Event: "Send Email"

Step 6 Map the fields
To: your email address (type it directly)
Subject: "New form response from " + [Name field from trigger]
Body: map Name, Email, Message fields from the trigger step

Step 7 Test the Zap
Click "Test step" — Zapier sends a real email using your last form response

Step 8 Publish
Click "Publish" > toggle ON > your Zap is live
Common mistake
Skipping the test step and going straight to Publish. Then you do not know if the email actually sends.
Right approach
Always click "Test step" before publishing. Zapier will send a real email from your last form response so you can verify the output looks right.

Before You Go Live: QA Checklist

Run every item before connecting to a client's real data.

7-point QA checklist
  1. Tested on dummy data, not real client data
  2. Verified the trigger fires correctly (check execution log)
  3. Confirmed the output arrived where expected (email, sheet, Slack)
  4. Checked that a failed run sends an alert rather than silently failing
  5. Named the automation clearly (what it does, not when you built it)
  6. Documented the trigger, actions, and any credentials in a shared doc
  7. Got client approval before connecting to live accounts
The Rule

A slow QA now means zero late-night fixes later. Test on dummy data every time, even for a "simple" change.

Day 5 · Research Automation

Stop Starting from Scratch

Manual research and automated research produce the same output. One takes 45 minutes; the other takes 2.

Without automation
Opens browser. Searches manually. Reads 6 tabs. Takes notes. Writes the brief. 45 minutes later: draft is ready.
With automation
Schedule fires. System pulls 5 sources. Claude summarizes into 3 bullets. Brief appears in inbox. 2 minutes: done.
The Standard

Automate the gathering. Keep the judgment. Your job shifts from collecting information to acting on it.

Build the Research Flow

Five nodes. Runs every Monday. You never have to think about it again.

Schedule
Mon 9:00 am
HTTP / Search
Pull 5 articles
Claude
Summarize + format
Deliver
Email or Slack
Tools that support this flow

Make.com or n8n for the workflow canvas. NewsAPI, Perplexity API, or Firecrawl for the search step. Claude API (via Make HTTP module) for the AI step. Gmail or Slack for delivery.

Your Prompt Is the Logic

In an automation the Claude node runs hundreds of times. A vague prompt gives vague output every single run.

Weak prompt
"Summarize this."

Result varies wildly. Sometimes a paragraph. Sometimes a list. Sometimes a refusal. Unusable at scale.
Strong prompt
"You are a research assistant. Given these article summaries, write exactly 3 bullets. Each bullet: one clear insight, max 20 words. Plain text only."

Consistent output every run. No surprises.
Automation-specific prompt rules
  • Always specify output format: bullet list, JSON, plain text, markdown
  • Set a word or character limit so the next step is not overwhelmed
  • Add a fallback instruction: "If there is no useful content, reply: NO_CONTENT"

When the AI Step Returns Nothing

Every AI step can fail silently. Add a fallback for each one.

What happensLikely causeFix
Empty outputPrompt too vague or input text too shortAdd output format + minimum word count to prompt; add NO_CONTENT fallback
Wrong formatForgot to specify structure in promptAdd explicit JSON or bullet format instruction; test with sample input
TimeoutAPI rate limit or network errorAdd error branch: if Claude step fails, send Slack alert with error detail
Hallucinated dataInput was ambiguous or contradictoryInclude source text in the prompt; ask Claude to cite the source inline
The Standard

Add a fallback for every AI step. If the step fails, send an alert. Silent failure is not an option.

Day 5 · Content & Social

Approved Content That Never Gets Posted

The bottleneck is not creation. It is the handoff from approved to live.

Without a pipeline
Content approved in Slack. Buried by notifications. Re-approved 3 days later. Copy-pasted to wrong platform. Posted late. No record of what went out.
With a pipeline
Content approved in Sheet. Automation fires. Posted to platform at scheduled time. Row updated to "Posted." Log is always accurate.
What this flow replaces

Manual copy-paste, missed posting slots, inconsistent timing, manual status updates, and the anxiety of "did that go out?"

Build the Content Flow

One approved row in your Sheet triggers the entire chain.

Sheet Trigger
New row = "Approved"
Filter
Status = Approved
Post Content
Buffer / Hootsuite
Update Row
Status to "Posted"
Sheet structure that makes this work

Columns: Post copy | Platform | Scheduled date | Status (Draft / Approved / Posted). The automation watches the Status column and acts only on "Approved."

Automate the Distribution, Not the Judgment

The rule

Never automate the approval step. The human reviews the content first, marks it approved, and only then does the automation act. This protects the client's brand and your relationship.

CategoryExamples
Automate freelyScheduling, formatting, cross-posting, logging, reminders, data entry
Keep humanApproval decisions, tone reviews, crisis responses, strategy choices
Gray areaFirst-draft creation (AI is fine), final copy (human must verify before it goes out)

The Real Cost of Manual Content Handoffs

Manual (without pipeline)
  • 45 min/week copy-pasting across platforms
  • Inconsistent posting times
  • Missed slots when you are busy
  • Manual status updates in a separate doc
  • No audit trail if something goes wrong
Automated (with pipeline)
  • 5 min/week: review and mark approved
  • Posts at the exact scheduled time
  • Nothing missed, even on busy days
  • Status auto-updates in the Sheet
  • Full log of every post: what, when, where
The Point

Your value to the client is not copy-paste speed. It is the judgment that decides what gets approved. Automate the rest.

Day 5 · Live Build Activity

Time to Build Yours

Before you build, you need to know what to build. Today you use an AI evaluator to find out.

The 3-step activity today
  1. List your tasks — write down 5 to 10 things you do repeatedly as a VA
  2. Run the Automation Evaluator — paste a prompt into Claude.ai and let it score your tasks
  3. Build the top pick — take the highest-scoring free task and build the automation right now
Why this matters

Most VAs guess what to automate. You will know. The evaluator scores every task on data, decision, and repetition — so you build the right thing first.

Day 5 · Live Build — Step 1

Run the Automation Evaluator

Open claude.ai (free account). Copy the prompt below, paste it in, then add your task list at the bottom where it says [YOUR TASKS HERE].

Automation Evaluator Prompt — copy this into Claude.ai
You are an automation evaluator. I will give you a list of tasks I do regularly as a VA.
Score each task out of 20 using these 4 dimensions (1-5 each):
- Repetitiveness: 1 = changes every time, 5 = identical every time
- Decision density: 1 = heavy judgment needed, 5 = no judgment, just execution
- Data clarity: 1 = messy unstructured input, 5 = clean predictable input
- Human touch: 1 = needs relationship or tone, 5 = no relationship needed

Add the 4 scores to get the Automation Potential Score (APS) out of 20.
Bucket each task:
🟢 16-20 = Automate Now
🟡 11-15 = AI-Assist (human in the loop)
🟠 6-10 = Streamline First
🔴 4-5 = Keep Human

For every 🟢 task: name the best FREE tool (Google Apps Script, Make.com free tier,
Zapier free tier, IFTTT, Notion Automations) and give numbered build steps.
For every 🔴 task: explain in one sentence why it must stay human.

End your response with TOP 3 QUICK WINS: highest score + simplest to build first.

Here are my tasks:
[YOUR TASKS HERE — list them one per line]
How to write your task list

One task per line. Be specific. Instead of "manage emails" write "forward all client inquiry emails to my tracker sheet." The more specific the task, the better the score.

Build It: Step by Step

Your turn
Follow these 5 steps with your chosen tool
  1. Open your tool (Zapier, Make, or Apps Script) and start a new workflow
  2. Set up the trigger: choose the event from your map (form submit, schedule, new row, etc.)
  3. Add the first action: connect the next step and map the data fields
  4. Add any filters or conditions from your "only if" question
  5. Add the final action (send, save, post, log) and turn on the automation
Golden rule for live builds

Build one step. Test one step. Do not add the next step until the current one works. The most common mistake is building five steps and then not knowing which one broke.

VA Standards: Before You Hand It Over

Professional standards checklist
  • Name the automation clearly: what it does, not when you built it ("Weekly Client Report" not "Automation 3")
  • Document the trigger, actions, and any API keys in a shared doc the client can access
  • Test on your own data before connecting to the client's live accounts
  • Tell the client what changed and how to pause or turn it off
  • Schedule a monthly review: check that triggers still fire and outputs still look right
Common mistakeWhat goes wrongFix
No error alertAutomation breaks silently; nobody notices for daysAdd an "on error: send me a Slack/email alert" branch to every flow
Real data in testsDuplicate emails sent, corrupted recordsAlways use dummy data in test mode before going live
No client sign-offClient did not want that automatedShow the flow diagram and get written approval before connecting
